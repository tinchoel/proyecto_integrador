Proyecto Integrador - Sistema de Facturación QA

Este proyecto es una aplicación Java modular desarrollada con Maven que simula un sistema de gestión y análisis de resultados de pruebas automatizadas dentro del dominio de facturación. Su objetivo principal es procesar archivos CSV que contienen resultados de test, validar su formato y generar reportes con estadísticas de ejecución.

Estructura del Proyecto

La aplicación sigue una arquitectura en capas, organizada en los siguientes paquetes:

  • com.qa.facturacion.model: Contiene las clases de modelo de datos que representan los resultados de los tests (TestCase) y su estado (TestStatus).
  • com.qa.facturacion.exception: Define excepciones personalizadas utilizadas para manejar errores específicos durante la lectura y validación de archivos CSV.
  • com.qa.facturacion.io: Proporciona clases de entrada/salida, incluyendo la lectura de datos desde archivos CSV (CsvTestReader) y la generación de reportes de salida (ReportGenerator).
  • com.qa.facturacion.service: Implementa la lógica de negocio para el cálculo de estadísticas (TestStatistics) y el modelado del reporte completo (TestReport).
  • com.qa.facturacion: Contiene la clase principal App, punto de entrada de la aplicación.

Flujo General de Ejecución

  1. El usuario ejecuta la aplicación pasando como parámetros la ruta del CSV y el directorio de salida.
  2. La clase CsvTestReader lee el archivo, validando cada línea y creando instancias de TestCase.
  3. Los datos válidos se procesan por TestStatistics para generar un resumen con totales y porcentajes.
  4. ReportGenerator produce un archivo de salida con las estadísticas y los errores detectados.
  5. Finalmente, App muestra el resultado por consola y registra la ejecución con SLF4J.

Dependencias

  • JUnit 5 – Framework de pruebas unitarias.
  • Apache Commons CSV (opcional) – Utilizado para parsear archivos CSV.
  • SLF4J + SimpleLogger – Sistema de logging ligero.

Ejemplo de Ejecución

java -jar proyecto-integrador-1.0-SNAPSHOT.jar tests-valid.csv salida/ --ignorar-cabecera

Convenciones y Buenas Prácticas

  • Separación de responsabilidades por paquete.
  • Métodos pequeños y fácilmente testeables.
  • Uso de Optional y Streams para un código más expresivo.
  • Documentación completa con Javadoc.
  • Excepciones específicas y controladas para errores de entrada.
